### (1) 大手自動車企業のDataHubアプリケーション

| 項目 | 内容 |
|------|------|
| 雇用形態 | 正社員 |
| 在籍期間 | 2023/12 ~ 現在 |
| チーム | PM 1名 / PMO 1名 / バックエンドエンジニア 3名 |
| ステークホルダー | 外部連携システム 5件 / ベンダー 2社（CloudOps / DevOps） |
| 言語・FW | Java / Micronaut |
| インフラ | AWS（ECS, Fargate, Batch, EC2, ALB, Aurora, Lambda, SNS, SQS, S3, CloudWatch, Kinesis Data Stream, Kinesis Firehose）/ Azure（EventHubs） |
| ツール | Jenkins / BitBucket / Confluence / Jira / ServiceNow / JUnit |
| 業界 | 自動車業界 |

**プロダクト概要**：データ流入元から届く大量のデータに対して、後続システムから要求される適切な計算や変換処理を行い、データを連携する DataHub アプリケーション

**チームの特徴**：ITコンサルタントのチームとして他社アプリケーション開発に参画。非エンジニアとの接点が多く、わかりやすい説明が求められる少人数チーム。

#### 課題と成果

**1. パフォーマンスを追求したAWS/アプリケーション設計・実装** — 開発者＆チーム内テックリード
> **課題**：約数千Requests/secの大量データを短時間で処理するためのAWS設計・アプリケーション設計が必要

- AWS設計およびアプリケーションの設計・開発を担当し、チーム内エンジニアのレビューも実施
- Cloudレベル：ECS/Fargateを用いたタスク分散、Pull型/Push型処理の適切な使い分けによりパフォーマンス要件を充足
- アプリケーションレベル：並行処理や高速なデータ型（Nodeインターフェース）を活用し処理速度を向上

**2. 設計からリリースまでのプロジェクトマネジメント** — 一人複数役の対応
> **課題**：少人数チームのため、一人で複数の役割を担うことが求められる環境

- 開発と並行して、設計からリリースまでのスケジュール管理、クライアント・外部連携システムとの調整、ベンダー管理を実施

**3. すべてのテスト工程を担当** — テスト専門家不在への対応
> **課題**：大規模テスト実施の中、チームにテスト工程の専門家がいない状況

- JUnit単体テストから、他システムとの総合テスト・性能テストまで全工程を担当（テスト要件取りまとめ→仕様書作成→実施）
- jMeterを用いたEventHubsへの大量データ投入など、テストツールの学習・導入・実施まで対応

**4. 海外ベンダーとの英語コミュニケーション**
> **課題**：CloudOps/DevOpsベンダーと英語でのやり取りが必要

- AWS設計をもとにCloudOpsと英語テキストベースで連携し、Cloud環境を構築

#### 工夫したこと
- **クローズドクエスチョンを意識した技術相談**：事前にPOCを行い、メリット/デメリット付きの複数選択肢と推奨案を用意。クライアントの意思決定負荷を軽減（例：リカバリーAPI拡張、Cloud構成など）
- **運用を考慮した技術調査**：運用フェーズの方が長いことを意識し、jMeterのMaster/Slave構成やリカバリーツール作成など、運用で有用な選択肢を提供
- **現状説明資料の随時更新**：多数のステークホルダーが存在する大規模プロジェクトで、常にステータスが把握できる資料を整備

| 項目 | 内容 |
|------|------|
| 雇用形態 | 正社員 → 準委任（離職後、準委任として継続運用） |
| 在籍期間 | 2023/02 ~ 現在 |
| チーム | PM 1名 / 営業 1名 / デザイナー 1名 / エンジニア 1名 |
| ステークホルダー | クライアント 3社 |
| 言語・FW | TypeScript / Next.js |
| インフラ | Vercel（アプリケーション）/ PlanetScale（DB） |
| ツール | GitHub / GitHub Actions |
| 業界 | フランチャイズ業界 |

**プロダクト概要**：広告代理店である自社が運用するLPにて発生した資料請求情報を管理するアプリケーション。フランチャイズオーナーが資料請求者の情報を閲覧し、面談申込メールを送付可能。フランチャイズ加盟率向上を目的とする。

**チームの特徴**：PM・デザイナー・営業・エンジニア各1名のミニマム構成での新規開発。社内初のアプリケーション開発であり、開発ナレッジがない状態からのスタート。

#### 課題と成果

**1. フルスタックでの対応** — 唯一のエンジニアとして全領域を担当
> **課題**：スタートアップフェーズでエンジニアは自分のみ。フロントからインフラまで一人で対応する必要あり

- Java/AWS経験をベースに、Next.jsやマネージドサービス（Vercel, PlanetScale）を新たに学習
- ChatGPTやGitHub Copilotを活用し、一人での開発速度を向上

**2. ビジネスフローのキャッチアップ＆仮デザイン制作** — 既存事業への適合
> **課題**：既存事業を推進するアプリのため、事業理解と設計への落とし込みが必要

- 積極的にビジネスフローを拾い、仕様の仮決め・Figmaでの仮デザイン作成まで実施
- デザインベースで仕様検討が可能になり、仕様確定・本デザイン依頼がスムーズに進行

**3. 採用・チームづくりを見据えた技術選定** — モダン言語での統一
> **課題**：メンバー増加を見据え、採用に有利かつ育成コストが抑えられる技術選定が必要

- フロントエンド・バックエンドをTypeScriptで統一し、全エンジニアが全領域を見れる体制を構築
- Vercel/PlanetScaleなどのマネージドサービスを採用し、アプリケーション開発に注力できる環境を整備

#### 工夫したこと
- **UX品質の基礎徹底**：BtoBアプリでもユーザ体験を重視し、メモ化やCLS抑制などの基礎を徹底
- **自動化による開発集中**：Prettier/ESLint/CI/CD（Vercel + GitHub Actions）を積極導入し、少人数でも設計・開発に集中できる環境を構築

### (3) 大規模動画配信サービスのフロントエンド開発

| 項目 | 内容 |
|------|------|
| 雇用形態 | 正社員 |
| 在籍期間 | 2022/06 ~ 2023/01 |
| チーム | PM 1名 / スクラムマスター 1名 / エンジニア 8名 |
| ステークホルダー | クライアント 3社 |
| 言語・FW | TypeScript / Next.js |
| インフラ | AWS（ECS, Fargate, ALB） |
| ツール | GitLab |
| 業界 | 映像業界 |

**プロダクト概要**：推定ユーザ数500万人の動画配信サービスのフロントエンドアプリケーション（既存アプリの移行先として新規開発）

**チームの特徴**：SIerとして他社のフロントエンド開発に参画。フロントエンド開発チームだけで5チーム存在し、他チームとの密な連携が必要。

#### 課題と成果

**1. フロントエンド開発者＆チームリーダー** — 短期間での集中開発
> **課題**：仕様決定・バックエンド開発が並行する中、開発期間5ヶ月でリリース可能な状態に仕上げる必要あり

- Next.jsを用いたフロントエンド開発（条件分岐による複数APIコール、フォロー機能等の実装）
- チームメンバーのコードレビュー、バックエンドとの調整を担当
- 他チームやAPI開発側と積極連携し、画像出力コンポーネントの共通化や負荷の少ないAPI設計・実装に貢献

**2. 技術者育成者＆レビュアー** — モダンフロントエンドの技術者不足への対応
> **課題**：自社内にReact.js/Next.js経験者がおらず、技術力向上が必要

- 配属決定時に先行してReact.js/Next.jsを学習し、メンバー向け育成プログラムを作成・実施
- useStateとuseEffect、コンポーネント分割を軸にした理解度テスト（1画面実装＋レビュー）で即戦力化
- 初回スプリントからクライアント満足のクオリティで実装を実現

#### 工夫したこと
- **モダンフロントエンド基礎講義の実施**：実装優先でおざなりになりがちな基礎（CSR/SSR、Reactレンダリングフローなど）を調査し、チーム向けに講義を実施。パフォーマンス低下や保守性低下を未然に防止
- **共通コンポーネントの優先着手**：他チームのスプリントレビューや開発者会議から課題を拾い、共通コンポーネントをいち早く作成。大規模開発の開発速度と品質向上に貢献
